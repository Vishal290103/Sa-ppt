Concept

What is a digital banking platform?

Definition
  • Banking system that delivers services through mobile and web channels.
  • Lets users check balances, transfer money, pay bills, and manage cards without visiting a branch.
  • Built for 24/7 access, speed, and self-service.
Why banks use it
  • Lower operating cost than branch-heavy banking.
  • Faster onboarding and better customer experience.
  • Easier scaling through APIs, automation, and cloud-ready services.

Architecture

Reference system architecture

1. Mobile app
Customer UI for login, balances, transfers, bill pay, notifications.
2. API gateway
Routes requests, applies rate limits, logging, and access control.
3. Auth service
Handles passwords, biometrics, OTP, sessions, and device checks.
4. Banking services
Accounts, payments, cards, alerts, fraud rules, customer profile.
5. Core systems
Core banking, payment rails, KYC, ledger, settlement and audit trail.
Design choice
Microservices improve scalability and fault isolation.
Integration
APIs connect mobile channels with banking back end.
Reliability
Monitoring and logs help track every banking action.
Compliance
Auditability supports regulation and fraud investigation.

Security

Secure authentication model

Main controls
  • Passwords or PINs for basic identity verification.
  • Biometric login such as fingerprint or face unlock on trusted devices.
  • OTP or step-up verification for high-risk actions.
  • Session management, device checks, and fraud-triggered re-authentication.
Security goal
  • Make login easy for normal users but harder for attackers.
  • Reduce phishing, account takeover, and unauthorized transactions.
  • Protect both account access and card/payment actions.
  • Maintain audit logs for compliance and dispute handling.

Processing

How transaction processing works

1. Request
User starts transfer or payment from mobile app.
2. Validation
System checks session, balance, limits, and account details.
3. Risk check
Fraud rules inspect device, amount, pattern, and beneficiary.
4. Execution
Payment engine sends instruction to core banking or payment rail.
5. Confirmation
Ledger updates, receipt generated, alert pushed to user.
Why this layer matters
A good transaction engine prevents duplicates, records audit trails, handles failures safely, and gives customers real-time confirmation.

  • Using UPI, customers could manage external bank accounts, check balances, and send or receive payments.

DBS security

How DBS strengthened trust

Customer protection
  • Payment Controls lets users manage card limits and lock risky transaction types in real time.
  • In 2025 DBS added a “Mobile wallets” toggle turned off by default to stop unauthorized card addition.
  • The process also required an OTP step, creating a deliberate pause before wallet linking.
Architecture lesson
  • Security should be built inside the app, not added only after fraud happens.
  • Self-service controls improve both usability and fraud prevention.
  • Authentication and transaction controls must work together as one design.
